Project Structure for Web Applications

Posted by Patrick Steele-Idem on Wednesday, Apr 30th, 2014   Web Development

Web developers have come up with various strategies for organizing their web application project files. One of the most common schemes is to create top-level folders for each different resource type--thus forking files into different top-level folders based on resource type. That is, there will be a top-level folder that contains all JavaScript files, another folder for CSS files and another folder for images and so on. However, I firmly believe that if you fork files by resource type in your web application then you are doing it wrong. This pervasive strategy bothered me so much that I decided to write a blog so other developers would not fall into the same trap.

First off, I think we all agree that is important to group related files. However, if you fork by resource type then you are essentially saying the most important grouping is the resource type. I propose a different solution, and that is to organize your files so that files that are used together are grouped together. Therefore, if you have a UI component, for example, that consists of a JavaScript file, a CSS file, an HTML template, some images and a properties file then I propose that you put all of the files in the same directory.
